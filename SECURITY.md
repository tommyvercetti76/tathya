# Security policy

**Reporting a vulnerability:** please use GitHub's private vulnerability reporting on this repository (Security → Report a vulnerability). If that is unavailable, open a plain issue saying only "security contact requested" — do **not** include details publicly — and a maintainer will provide a private channel.

Scope: the web app, the detection pipeline, the content-validation gate, and CI. Content *accuracy* disputes are not security reports — use the issue templates; that pipeline is public by design.

Please also read the full threat model and admin hardening handbook: [docs/SECURITY.md](docs/SECURITY.md).
